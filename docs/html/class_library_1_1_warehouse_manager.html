Detailed Description

El propósito de esta clase es "quitar" al admin la responsabilidad de administrar los depósitos, si bien por UserStory el admin es el encargado de esto, para evitar el acoplamiento se crea un administrador de depósitos que maneja el admin. La intención con el bot es utilizarla con singleton para que sea una única instancia y todos los usuarios manejen la misma lista de depósitos ubicada en warehouses. Esta clase cumple con SRP porque su responsabilidad es manjear los depósitos y lo que ocurre con los items que tiene dentro A su vez, cumple con el patrón Expert porque es esta clase conoce las ventas porque sabe que items ingresan y salen de los depositos.
